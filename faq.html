Why are the version numbers so ****ed??

The first number, 2.

Originally, the mod was a fork of Beepbox.
deathmodereal realised that this would limit the amount of things he could add to the mod, so he converted his changes to Ultrabox.
2 stands for the first revamp (the conversion to UltraBox), which is fork 2.

The second number, V82.

This mod's versions work in a pretty weird way, but here's the explanation.

Every time a file NOT in the website folder of the source code gets edited (all of the song editing things and such), a command would need to be run to affect the actual webpage (which will be reffered to as its filename for convenience sake, index.html).
This command turns the editor, synth, and player folders into .js files which index.html would use to make the song editor work.
Now, the version number increases every time the command is run AND index.html is opened to test the changes, which is why the version number is so high and why some updates have the same version number (such as V50 Quotes).
